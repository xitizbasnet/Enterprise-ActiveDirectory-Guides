## How to Prevent Domain Users from Clearing Browsing History in Microsoft Edge via GPO (Windows Server 2025)

## 📌 Overview

This document explains how to prevent domain users from clearing browsing history and download history in **Microsoft Edge** using **Group Policy Object (GPO)** in a Windows Server 2025 environment.

The configuration restricts users from deleting browser data such as:

- Browsing history  
- Download history  

---

## 🖥️ Client-Side Observation (Before GPO Configuration)

### 🔍 Steps to Verify Default Behavior on Client Computer

1. Go to the **client computer**
2. Open **Microsoft Edge**
3. Click on the **3-dot menu (⋮)** on the top-right corner near the user profile
4. Select **History**
5. Click the **Delete (trash/bin) icon**

### 📋 Pop-up Window: "Delete Browser Data"

You will see a confirmation window titled:

> **Delete browser data**

The following options are pre-selected with tick marks:

- Browsing history  
- Download history  

### 🔘 Action Available

- A **"Clear now"** button is available to delete selected data

---

## 🧭 Server-Side Configuration (GPO Setup)

### 🖥️ Open Group Policy Management

1. Go to **Server Manager**
2. Navigate to:
```

Tools > Group Policy Management

```

3. Expand your domain:
```

xitiztechservices.local (example domain)

```

4. Navigate to the target **Organizational Unit (OU)**:
```

OU > Domain_Users

```

---

### ➕ Create and Link New GPO

1. Right-click the OU (**Domain_Users**)
2. Select:
```

"Create a GPO in this domain, and link it here..."

```

3. Enter GPO name:
```

Deny User clear history in Microsoft Edge

```

4. Click **OK**

---

## ⚙️ GPO Configuration Settings

### ✏️ Edit the Created GPO

1. Right-click on:
```

Deny User clear history in Microsoft Edge

```
2. Select **Edit**

---

### 📁 Navigate to Policy Path

Go to:

```

User Configuration
└── Policies
└── Administrative Templates
└── Microsoft Edge

```

---

### 🔎 Configure Policy Setting

1. Locate the policy:
```

Enable deleting browser and download history

```

2. Double-click the policy

3. Set the option to:

```

❌ Disabled

````

---

### 💾 Apply Changes

- Click **Apply**
- Click **OK**

---

## 🔄 Update Group Policy

### 🖥️ On Server

Open **PowerShell (Run as Administrator)**:

```powershell
gpupdate /force
````

---

### 💻 On Client Computer

1. Open **PowerShell (Run as Administrator)**
2. Run:

```powershell
gpupdate /force
```

3. Restart the computer

---

## 🌐 Verification on Client System

### 🔍 Microsoft Edge Behavior After Policy Application

1. Open **Microsoft Edge**
2. Click on **3-dot menu (⋮)**
3. Navigate to:

   ```
   History
   ```
4. Click the **delete icon**

### 📋 Observed Change

The **Delete browser data** popup appears, but:

* The ability to clear **browsing history** is restricted (controlled by policy)
* The expected behavior reflects that deletion options are now governed by GPO restrictions
* Users are prevented from freely clearing browser history depending on policy enforcement

---

### 🔍 Google Chrome (Client Side)

3. Go to:

   ```
   History
   ```
4. Click the **delete icon**

### 📋 Pop-up Window: "Delete browsing data"

* Browsing history

### 🔘 Action Available

* A **"Clear now"** button is visible for clearing data
* Behavior may vary depending on domain policies or browser settings

---

## 📌 Summary

By configuring the following GPO setting:

```
User Configuration > Administrative Templates > Microsoft Edge > Enable deleting browser and download history = Disabled
```

Administrators can effectively restrict domain users from clearing browsing and download history in Microsoft Edge across managed systems.

---

## 🛠️ Notes

* Ensure GPO is linked to the correct OU
* Use `gpupdate /force` for immediate policy application
* Restart client machines if policy does not apply instantly
* Test changes on a pilot user group before full deployment
