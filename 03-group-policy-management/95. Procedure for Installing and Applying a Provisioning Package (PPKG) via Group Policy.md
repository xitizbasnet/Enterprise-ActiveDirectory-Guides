# 1 📘 Introduction

## 1.1 Abstract

This document provides detailed step-by-step procedures for installing and applying a Windows Provisioning Package (PPKG) through Group Policy Management in an Active Directory (AD) environment.
The purpose of this procedure is to automate deployment and ensure that all AD users receive the provisioning package consistently and securely.

---

# 2 ⚙️ Procedure

---

## 2.1 🖥 Accessing the Active Directory Server

1. Log in to the Active Directory (AD) server.
2. Open the Run dialog and type: `mstsc /admin`
   This command opens a Remote Desktop Connection session with administrative privileges.
3. In the Computer field, enter the IP address: `192.168.1.10`

---

## 2.2 🔐 Login Credentials

1. Use the following credentials to access the server:

| Username      | Password            |
| ------------- | ------------------- |
| Administrator | ******************* |

2. Click OK to connect.

---

## 2.3 📁 Creating and Sharing the System Folder

1. Navigate to the D: drive on the server.
2. Create a new folder named **sysadmin**.
3. Share this folder with **Everyone** and **Authenticated Users**, granting **Full Control (read and write)** permissions. This ensures all users can access the required files during deployment.

---

## 2.4 📦 Preparing the Provisioning Package File (PPKG)

1. Within the shared sysadmin folder, place the file named: **gtosotipoc.ppkg**
   (Note: This provisioning package will be installed automatically for AD users.)

---

## 2.5 💻 Creating the PowerShell Installation Script

Create a new PowerShell script named **Install-PPKG.ps1** in the sysadmin folder. Use the following code:

### Starting Code:

```powershell
# Install-PPKG.ps1
$ppkgPath = "\\192.168.1.10\sysadmin\gtsotipoc.ppkg"
$logPath = "C:\Windows\Temp\PPKG_Install.log"

# Create log directory if it doesn't exist
if (!(Test-Path -Path (Split-Path $logPath))) {
New-Item -ItemType Directory -Path (Split-Path $logPath) -Force | Out-Null
}

# Start logging
Add-Content -Path $logPath -Value "$(Get-Date -Format 'yyyy-MM-dd HH:mm:ss') - Starting provisioning package installation."

try {
Install-ProvisioningPackage -PackagePath $ppkgPath -QuietInstall -ErrorAction Stop
Add-Content -Path $logPath -Value "$(Get-Date -Format 'yyyy-MM-dd HH:mm:ss') - Successfully installed provisioning package from $ppkgPath."
} catch {
Add-Content -Path $logPath -Value "$(Get-Date -Format 'yyyy-MM-dd HH:mm:ss') - Failed to install provisioning package. Error: $_"
}
```

### Ending Code.

This script installs the provisioning package silently and generates a log file under `C:\Windows\Temp` to record the installation status.

---

## 2.6 🧩 Configuring Group Policy Management

1. Open Group Policy Management from the Windows Start menu.
2. New policy named as: **Deploy PPKG Nov 7, 2025**
3. Apply the PPKG configuration to the relevant domain users:
   • Example: gg user and golyangroup.local
4. Navigate to the following path in the Group policy Editor:
   • Computer configuration > policies > windows setting > scripts (Startup/Shutdown) > startup
5. Click Add to include a new startup script.

---

## 2.7 📜 Adding the PowerShell Script

When prompted, add the script.

6. Adding the script.

### 📊 Table 2: Script

| Script Name    | Script Parameter                                                        |
| -------------- | ----------------------------------------------------------------------- |
| powershell.exe | -ExecutionPolicy Bypass -File "\192.168.1.10\sysadmin\Install-PPKG.ps1" |

7. Click OK to save the configuration.
   • After saving, verify that the script entry appears correctly in the policy settings.
8. Click OK
9. The final result will be seen as in the policy

---

## 2.8 🚀 Applying and Enforcing the Policy

1. Right-click on the created Group Policy.
2. Select **Enforced**.
3. Open Command Prompt as an Administrator.
4. Run the following command to apply the policy immediately:
   • `gpupdate /force`
5. Once the command completes successfully, restart the AD client computers to apply the new policy.

---

# 3 📘 Conclusion

By following the above procedure, the provisioning package is successfully deployed to all targeted Active Directory users using Group Policy Management.

This automated deployment method ensures consistent application of configuration packages across the organization, reducing manual intervention and ensuring system uniformity.
