# How to Remotely Generate Full Hard Disk Space Report for Domain Computers on Windows Server 2025

## 📘 Overview

This guide explains how to remotely generate a **full hard disk space usage report** for all **domain-joined computers** using **Windows Server 2025** and **PowerShell**.

The generated report includes:

* Computer Name
* Total Disk Size
* Drive Letter
* Total Space
* Used Space
* Free Space
* Online/Offline Status

The final report is exported as a **CSV file** and can be opened using **Microsoft Excel**.

# 💻 Step 5: Run the PowerShell Script

## Original Command

```powershell
$computers = Get-ADComputer -Filter * | Select-Object -ExpandProperty Name

$result = foreach ($pc in $computers) {

    try {
        $disks = Get-CimInstance -ComputerName $pc -ClassName Win32_LogicalDisk -Filter "DriveType=3" -ErrorAction Stop

        $totalDiskSize = ($disks | Measure-Object -Property Size -Sum).Sum

        foreach ($d in $disks) {

            $total = $d.Size
            $free  = $d.FreeSpace
            $used  = $total - $free

            [PSCustomObject]@{
                Computer       = $pc
                TotalDiskGB    = [math]::Round($totalDiskSize / 1GB,2)
                Drive          = $d.DeviceID
                TotalGB        = [math]::Round($total / 1GB,2)
                UsedGB         = [math]::Round($used / 1GB,2)
                FreeGB         = [math]::Round($free / 1GB,2)
                Status         = "Online"
            }
        }
    }
    catch {
        [PSCustomObject]@{
            Computer       = $pc
            TotalDiskGB    = "N/A"
            Drive          = "-"
            TotalGB        = "-"
            UsedGB         = "-"
            FreeGB         = "-"
            Status         = "Offline"
        }
    }
}

# Export CSV
$path = "C:\Reports\Disk_Final_Report.csv"
New-Item -ItemType Directory -Path "C:\Reports" -Force | Out-Null

$result | Export-Csv -Path $path -NoTypeInformation -Encoding UTF8

Write-Host "Report Created: $path" -ForegroundColor Green
```

# 🔒 Requirements

Before running the script, ensure the following:

* The server is joined to the domain.
* Remote management is enabled on client computers.
* PowerShell has administrative privileges.
* The `ActiveDirectory` module is installed.
* Firewall rules allow remote CIM/WMI access.

---

# 🛠️ Troubleshooting

## ❌ Computers Showing as Offline

Possible causes:

* Computer is powered off
* Network connectivity issue
* Firewall blocking WMI/CIM requests
* Remote management disabled

---

## ❌ `Get-ADComputer` Command Not Working

Install the Active Directory module:

```powershell
Install-WindowsFeature RSAT-AD-PowerShell
```

---

# 📌 Notes

* The script scans all domain computers automatically.
* Only local hard disks (`DriveType=3`) are included in the report.
* The report is saved in CSV format for easy analysis and filtering.
* The script creates the `C:\Reports` folder automatically if it does not exist.
