# Group Policy Update Deployment via PowerShell

## 📌 Overview

This document provides instructions for triggering a **Group Policy Update (GPUpdate)** across all Active Directory domain computers using **PowerShell** on the AD server.

---

# 🖥️ Procedure

## 1. Open the AD Server

* Log in to the Active Directory (AD) server with an account that has administrative privileges.

---

## 2. Launch PowerShell as Administrator

1. Open the **Start Menu**.
2. Search for **PowerShell**.
3. Right-click **Windows PowerShell**.
4. Select **Run as administrator**.

> ⚠️ Administrative privileges are required to execute remote Group Policy updates.

---

## 3. Execute the PowerShell Script

Paste the following script into the PowerShell window:

```powershell
$Computers = Get-ADComputer -Filter * | Select-Object -ExpandProperty Name

foreach ($Computer in $Computers) {

    if (Test-Connection $Computer -Count 1 -Quiet) {

        try {

            Invoke-GPUpdate -Computer $Computer -Force -RandomDelayInMinutes 0

            Write-Host "$Computer : GPUpdate Triggered" -ForegroundColor Green

        }

        catch {

            Write-Host "$Computer : Failed" -ForegroundColor Red

        }

    }

    else {

        Write-Host "$Computer : Offline" -ForegroundColor Yellow

    }
}
```

---

## 4. Run the Script

* Press **Enter** to execute the script.

---

# 📋 Script Functionality

| Function          | Description                                          |
| ----------------- | ---------------------------------------------------- |
| `Get-ADComputer`  | Retrieves all computer objects from Active Directory |
| `Test-Connection` | Checks whether the target computer is online         |
| `Invoke-GPUpdate` | Forces a Group Policy update on the remote machine   |
| `Write-Host`      | Displays execution status with color-coded output    |

---

# 🎨 Output Status Reference

| Status                  | Meaning                                          |
| ----------------------- | ------------------------------------------------ |
| 🟢 `GPUpdate Triggered` | Group Policy update was successfully initiated   |
| 🔴 `Failed`             | The update process encountered an error          |
| 🟡 `Offline`            | The target machine is unreachable or powered off |

---

# ⚠️ Requirements

* Active Directory PowerShell module installed
* Domain Administrator or equivalent privileges
* Remote machines must allow remote Group Policy updates
* Network connectivity between the server and client machines

---

# ✅ Expected Result

After execution:

* Online computers will receive an immediate Group Policy update request.
* Offline systems will be skipped and marked accordingly.
* Failures will be displayed for troubleshooting and review.

---

# 🛠️ Troubleshooting Tips

## If `Get-ADComputer` Fails

Verify the Active Directory module is installed:

```powershell
Get-Module -ListAvailable ActiveDirectory
```

Import the module if necessary:

```powershell
Import-Module ActiveDirectory
```

---

## If `Invoke-GPUpdate` Fails

Check:

* Firewall settings
* Remote Procedure Call (RPC) availability
* Administrative permissions
* Network connectivity

---

# 📚 Notes

> 💡 This script performs a bulk GPUpdate across all domain-joined computers returned by Active Directory. Use caution when executing in large production environments.
