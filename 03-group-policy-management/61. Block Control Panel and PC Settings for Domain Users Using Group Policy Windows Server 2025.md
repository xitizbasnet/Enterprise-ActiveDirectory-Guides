# 🔒 Block Control Panel and PC Settings for Domain Users

### Using Group Policy (Windows Server 2025)

![Windows Server](https://img.shields.io/badge/Windows%20Server-2025-blue)
![Active Directory](https://img.shields.io/badge/Active%20Directory-GPO-green)
![Category](https://img.shields.io/badge/Category-System%20Administration-orange)

---

# 📘 Overview

In an enterprise environment, it is important to restrict users from modifying critical system settings. Allowing unrestricted access to **Control Panel** and **Windows Settings** can lead to unauthorized configuration changes, security risks, or system instability.

This guide explains how to **block access to Control Panel and PC Settings for domain users using Group Policy Object (GPO)** in a **Windows Server 2025 Active Directory domain environment**.

After implementing this policy, users will not be able to open **Control Panel** or modify system settings.

---

# 🎯 Objective

The objective of this configuration is to:

* Prevent domain users from accessing **Control Panel**
* Restrict access to **Windows Settings**
* Maintain **centralized administrative control**
* Improve **system security and compliance**

---

# 🖥 Environment Requirements

Before implementing this configuration, ensure the following:

* Windows Server 2025 Domain Controller
* Active Directory Domain Environment
* Client computer joined to the domain
* Administrative privileges to manage Group Policy

Example domain used in this guide:

```
xitiztechservices.local
```

---

# 🔎 Step 1 — Verify Current Access on Client Computer

Before applying the restriction, verify that the client computer currently allows access to Control Panel and Settings.

1. First go to the client computer.
2. Check whether the **Control Panel** is opening or not.
3. Also check whether **Windows Settings** can be accessed.

**Assumption**

The client computer can access all the settings and Control Panel features and they can be used by the user.

---

# 🖥 Step 2 — Configure the Policy in Domain Controller

Now we will disable this access using **Group Policy** from the Domain Controller.

We are going to disable it via **GPO in the domain controller**.

---

# ⚙ Step 3 — Open Group Policy Management

1. Go to the **Server (Domain Controller)**.
2. Open:

```
Tools → Group Policy Management
```

3. Navigate to the domain.

Example:

```
xitiztechservices.local
```

4. Locate the Organizational Unit (OU).

Example:

```
Domain Users
```

5. Right-click the OU and select:

```
Create a GPO in this domain, and Link it here
```

6. Name the new Group Policy Object:

```
Block Control Panel and PC Settings GPO
```

7. Click **OK**.

8. Right-click the newly created GPO:

```
Block Control Panel and PC Settings GPO
```

9. Click **Edit**.

---

# 🔧 Step 4 — Configure the Policy Setting

Navigate to the following policy path:

```
User Configuration
   → Policies
      → Administrative Templates
         → Control Panel
```

Locate the following policy:

```
Prohibit access to Control Panel and PC Settings
```

Then perform the following actions:

1. Double-click the policy.
2. Select **Enabled**.
3. Click **Apply**.
4. Click **OK**.

This policy will block users from accessing both **Control Panel** and **Windows Settings**.

---

# 🔄 Step 5 — Update Group Policy on the Server

After configuring the policy, update the Group Policy on the server.

1. Open **PowerShell**.
2. Run it **as Administrator**.

Execute the following command:

```
gpupdate /force
```

This command forces the server to apply the latest Group Policy changes.

---

# 💻 Step 6 — Update Group Policy on Client Computer

Now apply the policy update on the client computer.

1. Open the **Client Computer**.
2. Search for **Command Prompt**.
3. Run it **as Administrator**.

Execute the following command:

```
gpupdate /force
```

Restart the client computer.

```
Restart the client PC
```

---

# ✅ Step 7 — Test and Validation

After rebooting the client system, verify whether the policy has been successfully applied.

### Test Control Panel

Open **Control Panel**.

Expected result:

A restriction message should appear:

```
"This operation has been cancelled due to restrictions in effect on this computer. Please contact your system administrator."
```

This confirms that the **Control Panel has been successfully blocked**.

---

### Test Windows Settings

Now test the **Windows Settings application**.

Click **Settings**.

Expected behavior:

* The Settings application may not open.
* Clicking it may give **no response**.

This indicates that **Windows Settings access is also restricted**.

---

# 🎉 Result

After implementing this policy:

✔ Control Panel access is blocked

✔ Windows Settings access is restricted

✔ Users cannot modify system configurations

✔ Administrative control is maintained within the domain environment

---

# 📌 Best Practices

For enterprise environments, it is recommended to:

* Apply the policy to **specific Organizational Units (OU)**
* Test the policy in a **test environment before production**
* Document all GPO changes for **audit and compliance**

This guide demonstrated how to **block Control Panel and PC Settings using Group Policy in Windows Server 2025**.

By implementing this configuration, IT administrators can prevent users from modifying system settings and maintain a **secure and controlled Active Directory environment**.
