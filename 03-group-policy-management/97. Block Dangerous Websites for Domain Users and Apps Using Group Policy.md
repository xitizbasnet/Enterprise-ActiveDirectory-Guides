# 🛡️ Windows Server 2025 Group Policy Guide

# 🚫 How To Block Dangerous Websites for Domain Users and Apps Using Group Policy

# 1 Purpose

🎯 This document outlines the procedure for blocking dangerous websites and malicious domains for domain users and applications using **Group Policy** in **Windows Server 2025**.

The configuration uses:

* 🛡️ Microsoft Defender Antivirus
* 🌐 Network Protection
* 🔒 Microsoft Defender Exploit Guard
* ⚙️ Group Policy Management

This setup helps improve:

* ✅ Network security
* ✅ Malware prevention
* ✅ Protection against phishing websites
* ✅ Domain-wide web protection policies

# 3 Open Group Policy Management

## ⚙️ Access Procedure

### Step 1 — Open Server Manager

Search for:

```text
Server Manager
```

---

### Step 2 — Open Group Policy Management

Navigate to:

```text
Tools > Group Policy Management
```

---

# 4 Create and Link New GPO

## 🏗️ Create New Group Policy Object

### Step 1 — Select Domain

In the server name:

```text
xitiztechservices.local
```

---

### Step 2 — Select Organizational Unit (OU)

Select an OU.

### Example:

```text
Domain User
```

---

### Step 3 — Create New GPO

Right click the OU.

Select:

```text
Create a GPO in this domain and link it here
```

---

### Step 4 — Configure GPO Name

#### a. GPO Name

```text
Block Domain user and app from website access dangerous webistes
```

---

### Step 5 — Edit GPO

Select:

```text
Edit
```

---

# 5 Configure Network Protection Policy

## 🌐 Block Dangerous Websites

Navigate to the following policy path:

```text
Computer Configuration > Policies > Administrative Templates > Windows Components > Microsoft Defender Antivirus > Microsoft Defender Exploit Guard > Network Protection
```

---

### Step 1 — Locate Policy

Search for the setting:

```text
Prevent users and apps from accessing dangerous websites
```

---

### Step 2 — Edit Policy

Double click to configure it.

---

### Step 3 — Enable Policy

Check mark on:

```text
Enabled
```

---

> 📌 Note:
>
> Enabled means:
>
> ```text
> Block: Users and applications will not be able to access dangerous domains
> ```

---

### Step 4 — Configure Options

Under the options section:

```text
Block
```

---

### Step 5 — Save Configuration

Click:

```text
Apply > OK
```

---

# 6 Configure Real-Time Protection Policy

## 🛡️ Enable Defender Real-Time Protection

On same policy path:

```text
Computer Configuration > Policies > Administrative Templates > Windows Components > Microsoft Defender Antivirus > Microsoft Defender Exploit Guard > Real-Time Protection
```

---

### Step 1 — Locate Policy

Search for the setting:

```text
Turn off real-time protection
```

---

### Step 2 — Edit Policy

Double click to edit it.

---

### Step 3 — Configure Policy

Click on radio button:

```text
Disabled
```

---

> 📌 Note:
>
> Setting this policy to **Disabled** ensures that Microsoft Defender Real-Time Protection remains active.

---

### Step 4 — Save Configuration

Click:

```text
Apply > OK
```

---

# 7 Configure Microsoft MAPS Policy

## 🌍 Configure Microsoft MAPS

On same policy path:

```text
Computer Configuration > Policies > Administrative Templates > Windows Components > Microsoft Defender Antivirus > Microsoft Defender Exploit Guard > MAPS
```

---

### Step 1 — Locate Policy

Search for the setting:

```text
Join Microsoft MAPS
```

---

### Step 2 — Edit Policy

Double click to edit it.

---

### Step 3 — Enable Policy

Click on radio button:

```text
Enabled
```

---

### Step 4 — Configure MAPS Option

Under the options section:

```text
Join Microsoft MAPS: Basic MAPS
```

---

### Step 5 — Save Configuration

Click:

```text
Apply > OK
```

---

# 8 Apply Group Policy Update

## 🔄 Force Group Policy Update

### Step 1 — Open PowerShell

Go to Windows Search.

Search for:

```text
PowerShell
```

Run as:

```text
Run as Administrator
```

---

### Step 2 — Update Group Policy

Run the following command:

```powershell
gpupdate /force
```

---

### Step 3 — Wait for Completion

Allow the Group Policy update process to complete successfully.
