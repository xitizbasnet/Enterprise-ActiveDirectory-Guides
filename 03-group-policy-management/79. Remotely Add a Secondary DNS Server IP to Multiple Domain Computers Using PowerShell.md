# 🖥️ How to Remotely Add a Secondary DNS Server IP to Multiple Domain Computers Using PowerShell

---

## 📌 Overview
This guide explains how to remotely configure an **Alternative (Secondary) DNS Server** on multiple domain-joined computers using **PowerShell** from an Active Directory (AD) server.

---

## 🧑‍💻 Prerequisites
- Domain-joined computers
- Administrative privileges on the AD server
- PowerShell remoting enabled on target machines
- Network connectivity between server and client machines

---

## 🔍 Step 1: Verify Current DNS Settings on Client Computer

### Example Client:
`Account_PC_01`

1. Open **Control Panel**
2. Set **View by:** `Large icons`
3. Navigate to **Network and Sharing Center**
4. Locate the network connection:
   - Example: `Ethernet0`
5. Click to view **Status**
6. Click **Properties**
7. Locate and select:
   - `Internet Protocol Version 4 (TCP/IPv4)`
8. Double-click to open settings

### 📡 DNS Configuration (Before)
```

Preferred DNS Server:  192.168.1.201
Alternative DNS: Empty

````

---

## ⚙️ Step 2: Prepare PowerShell Script

### 🧾 Original Code
```powershell
$computers = @("pc-name", "pc-name"

Invoke-Command -ComputerName $computers -ScriptBlock {

    $iface = Get-NetAdapter | Where-Object {$_.InterfaceAlias -eq "adabtername"}

    $current = (Get-DnsClientServerAddress -InterfaceIndex $iface.ifIndex).ServerAddresses

    if ($current -notcontains "8.8.8.8") {
        $new = @($current) + "8.8.8.8"
        Set-DnsClientServerAddress -InterfaceIndex $iface.ifIndex -ServerAddresses $new
    }
}
````

---

### ✏️ Modified Code (As per Client Domain Computers)

```powershell
$computers = @("AccountPC01", "ITPC11")

Invoke-Command -ComputerName $computers -ScriptBlock {

    $iface = Get-NetAdapter | Where-Object {$_.InterfaceAlias -eq "Ethernet0"}

    $current = (Get-DnsClientServerAddress -InterfaceIndex $iface.ifIndex).ServerAddresses

    if ($current -notcontains "8.8.8.8") {
        $new = @($current) + "8.8.8.8"
        Set-DnsClientServerAddress -InterfaceIndex $iface.ifIndex -ServerAddresses $new
    }
}
```

---

## 🖥️ Step 3: Execute Script from AD Server

1. Log in to the **AD Server**
2. Open **PowerShell** as **Administrator**
3. Paste the **modified script**
4. Press **Enter** and wait until execution completes

---

## 🔁 Step 4: Verify DNS Update on Client Computer

### Return to Client:

`Account_PC_01`

Repeat the navigation steps:

1. Open **Control Panel**
2. Set **View by:** `Large icons`
3. Go to **Network and Sharing Center**
4. Select connection:

   * `Ethernet0`
5. View **Status**
6. Click **Properties**
7. Open:

   * `Internet Protocol Version 4 (TCP/IPv4)`

---

### 📡 DNS Configuration (After)

```
Preferred DNS Server:  192.168.1.201
Alternative DNS: 8.8.8.8
```

---

## ✅ Result

The **Alternative DNS Server (8.8.8.8)** has now been successfully applied to the client machines remotely via the AD server using PowerShell.

---

## 📝 Notes

* Ensure the **interface name** (e.g., `Ethernet0`) matches the target systems.
* Confirm **PowerShell Remoting** is enabled (`Enable-PSRemoting`).
* Validate firewall rules allow remote execution.
* The script avoids duplication by checking if the DNS already exists.

---

## 🏁 Summary

This process allows centralized management of DNS configurations across multiple domain computers, improving efficiency and consistency in network administration.
