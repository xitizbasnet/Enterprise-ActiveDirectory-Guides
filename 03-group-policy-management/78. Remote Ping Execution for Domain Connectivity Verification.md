# 📡 Remote Ping Execution for Domain Connectivity Verification

## 📘 Overview
This document outlines the process for remotely running a ping test using PowerShell to verify domain connectivity between client computers and an Active Directory (AD) server.

---

## 🖥️ Environment Details

- **Domain Name:** `xitiztechservices.local`
- **Client Computers:**
  - `Account01PC`
  - `Reception01PC`
- **AD Server:**
  - `ADServer01`

---

## ⚙️ Step-by-Step Procedure

### 🔹 Step 1: Verify Hostname on Client Computer (Account01PC)

1. Go to the client computer.
2. Open search and type: **Windows PowerShell**.
3. Right-click and select **Run as Administrator**.
4. Execute the following command:

```powershell
hostname
````

**Expected Output:**

```
Account01PC
```

---

### 🔹 Step 2: Verify Hostname on AD Server

1. Go to the server.
2. Open search and type: **Windows PowerShell**.
3. Right-click and select **Run as Administrator**.
4. Execute the following command:

```powershell
hostname
```

**Expected Output:**

```
ADServer01
```

---

### 🔹 Step 3: Verify Hostname on Another Client Computer (Reception01PC)

1. Go to the client computer.
2. Open search and type: **Windows PowerShell**.
3. Right-click and select **Run as Administrator**.
4. Execute the following command:

```powershell
hostname
```

**Expected Output:**

```
Reception01PC
```

---

### 🔹 Step 4: Execute Remote Ping Test

1. Go to the client computer: **Account01PC**.
2. Open **Windows PowerShell** as Administrator.
3. Run the following command:

```powershell
Test-Connection -Source Reception01PC -ComputerName ADServer01 -Credential Domain\Administrator
```

4. Press **Enter**.

---

### 🔐 Authentication Prompt

* A credential window will appear.
* Enter the **Administrator username and password** for the AD Server.

---

## ✅ Expected Result

After successful execution and authentication, the output will display:

```
Source: Reception01PC
Destination: ADServer01
IPv4 Address: 192.168.1.201
IPv6 Address: Empty
Bytes: 32 
Time(ms): 0
```

---

## 📝 Notes

### 📌 Command Syntax

```powershell
Test-Connection -Source RemotePcname -ComputerName TargetComputer -Credential Domain\Admin
```

### 📌 Example Usage

```powershell
Test-Connection -Source Account01PC -ComputerName ADServer01 -Credential Domain\Administrator
```

### 📌 Parameter Explanation

* **`-Source`**
  👉 Specifies the computer from which the ping request is initiated.

* **`-ComputerName` (Target Computer)**
  👉 Specifies the destination computer (typically the AD Server).

* **`-Credential`**
  👉 Provides the necessary domain administrator credentials for authentication.

---

## 📎 Summary

This process allows administrators to:

* Remotely test network connectivity between domain-joined machines.
* Validate communication with the Active Directory server.
* Troubleshoot network or domain-related issues efficiently.
