# How to Block Ping Requests From All Domain Computers Except Admin Computer on Windows Server 2025

## 📌 Overview

This guide explains how to block ICMP Echo Requests (Ping) from all domain computers except a specific administrator computer on a Windows Server 2025 Domain Controller using Windows Defender Firewall with Advanced Security.

---

# 🖥️ Environment Details

| Component            | Value                     |
| -------------------- | ------------------------- |
| Domain Controller IP | `192.168.1.201`           |
| Admin Computer IP    | `192.168.1.11`            |
| Domain Name          | `xitiztechservice.local`  |
| Firewall Type        | Windows Defender Firewall |
| Protocol Used        | ICMPv4                    |

---

# ✅ Step 1: Verify Ping From Admin Computer

## 📍 On the Admin Computer

### Open Command Prompt as Administrator

1. Press `Win + R`
2. Type:

```cmd
cmd
```

3. Right-click **Command Prompt**
4. Select **Run as administrator**

---

## 🔍 Test Ping Connectivity

Run the following command:

```cmd
ping 192.168.1.201 -t
```

### ✅ Expected Result

Ping replies should appear continuously.

Example:

```cmd
Reply from 192.168.1.201: bytes=32 time<1ms TTL=128
```

This confirms that the admin computer can successfully ping the Domain Controller.

---

# 🔎 Step 2: Check Admin Computer IP Address

## 📍 Open Network Details

1. Open:

```text
Control Panel
→ All Control Panel Items
→ Network and Sharing Center
→ Ethernet0
→ Details
```

2. Locate the IPv4 Address.

### ✅ Admin Computer IP Address

```text
192.168.1.11
```

3. Click **Close**
4. Close the Control Panel

---

# ✅ Step 3: Verify Ping From Client Computer

## 📍 On the Client Computer

### Open Command Prompt as Administrator

1. Press `Win + R`
2. Type:

```cmd
cmd
```

3. Right-click **Command Prompt**
4. Select **Run as administrator**

---

## 🔍 Test Ping Connectivity

Run the following command:

```cmd
ping 192.168.1.201 -t
```

### ✅ Expected Result

Ping replies should appear.

This confirms that the client computer can currently ping the Domain Controller.

---

# 🔥 Step 4: Configure Firewall Rule on Windows Server 2025

## 📍 On the Domain Controller

Server:

```text
xitiztechservice.local
```

---

## 🛡️ Open Windows Defender Firewall Advanced Settings

Navigate to:

```text
Control Panel
→ Windows Defender Firewall
→ Advanced Settings
```

---

# ⚙️ Create New Inbound Rule

## 📌 Rule Configuration Steps

1. Right-click **Inbound Rules**
2. Select **New Rule**
3. Choose:

```text
Custom
```

4. Click **Next**

---

## 📌 Program Selection

Choose:

```text
All programs
```

Click **Next**

---

# 🌐 Protocol and Ports Configuration

Fill in the details as follows:

| Setting         | Value     |
| --------------- | --------- |
| Protocol Type   | ICMPv4    |
| Protocol Number | 1         |
| Local Ports     | All Ports |
| Remote Ports    | All Ports |

---

## 📌 Configure ICMP Settings

1. Under:

```text
Internet Control Message Protocol (ICMP) Settings
```

2. Click:

```text
Customize
```

---

## 📌 Enable Specific ICMP Types

1. Enable the radio button:

```text
Specific ICMP Types
```

2. Select and tick:

```text
Echo Request
```

3. Click **OK**

---

# 📍 Local IP Address Configuration

## Which Local IP Addresses Does This Rule Apply To?

1. Enable:

```text
These IP Addresses
```

2. Click **Add**

3. Enter:

```text
192.168.1.201
```

4. Click **OK**

---

# 📍 Remote IP Address Configuration

## Which Remote IP Addresses Does This Rule Apply To?

### Enable:

```text
These IP Addresses
```

---

# 🚫 Add First Blocked IP Range

1. Click **Add**
2. Enable:

```text
This IP Address Range
```

3. Enter:

```text
From: 192.168.1.1
To:   192.168.1.10
```

### 📝 Note

These IP addresses are blocked.

4. Click **OK**

---

# 🚫 Add Second Blocked IP Range

1. Click **Add**
2. Enable:

```text
This IP Address Range
```

3. Enter:

```text
From: 192.168.1.12
To:   192.168.1.254
```

### 📝 Note

These IP addresses are excluded from ping access.

4. Click **OK**

5. Click **Next**

---

# ⛔ Action Configuration

Select:

```text
Block the connection
```

Click **Next**

---

# 🌍 Profile Configuration

Tick all profiles:

* ✅ Domain
* ✅ Private
* ✅ Public

Click **Next**

---

# 🏷️ Rule Name

Enter the rule name:

```text
block ping
```

Click **Finish**

---

# ✅ Firewall Rule Successfully Created

The ICMP blocking rule is now successfully configured on the Windows Server 2025 Domain Controller.

---

# 🧪 Step 5: Test From Client Computer

## 📍 On the Client Computer

Open Command Prompt as Administrator and run:

```cmd
ping 192.168.1.201 -t
```

---

## ❌ Expected Result

Ping requests should now be blocked.

Example:

```cmd
Request timed out.
```

This confirms that client computers can no longer ping the Domain Controller.

---

# 🧪 Step 6: Verify Admin Computer Still Has Access

## 📍 On the Admin Computer

Open Command Prompt as Administrator and run:

```cmd
ping 192.168.1.201 -t
```

---

## ✅ Expected Result

Ping replies should continue successfully.

Example:

```cmd
Reply from 192.168.1.201: bytes=32 time<1ms TTL=128
```

This confirms that the admin computer is still allowed to ping the Domain Controller.

# ✅ Final Result

| Device                          | Ping Access |
| ------------------------------- | ----------- |

---

# 📚 Summary

This configuration uses a custom inbound firewall rule in Windows Defender Firewall to:

* Block ICMP Echo Requests from all specified client IP ranges
* Allow only the administrator computer to ping the Domain Controller
* Improve server security and network access control

---

# 🔐 Security Recommendation

For enterprise environments, consider:

* Using Group Policy (GPO) to centrally manage firewall rules
* Restricting ICMP traffic only where necessary
* Documenting allowed management IP addresses
* Regularly auditing firewall configurations
