# 🚀 Windows Deployment Services (WDS) 

### Deploy Windows 11 using Windows Server 2025

This guide explains how to install and configure **Windows Deployment Services (WDS)** on **Windows Server 2025** to deploy **Windows 11** over the network using **EFI / PXE Boot**.

This documentation is designed for **System Administrators and IT Infrastructure Teams** managing automated Windows deployments.

# 🧭 Overview

**Windows Deployment Services (WDS)** allows administrators to deploy Windows operating systems across multiple machines using network-based installation.

With WDS you can:

✔ Deploy Windows without USB/DVD

✔ Automate operating system installation

✔ Manage centralized OS images

✔ Reduce manual installation time

✔ Deploy OS across enterprise networks


---

# ⚙️ Infrastructure Prerequisites

Before installing WDS, ensure the following services exist in your environment.

| Requirement       | Description                         |
| ----------------- | ----------------------------------- |
| Domain Controller | Active Directory must be configured |
| DHCP Server       | Required for IP assignment          |
| DNS Server        | Required for domain name resolution |
| NTFS Partition    | Required for WDS image storage      |

# 📦 Installing Windows Deployment Services

### Step 1 — Open Server Manager

```
Server Manager → Dashboard
```

Click:

```
Add Roles and Features
```

---

### Step 2 — Select Installation Type

Choose:

```
Role-based or feature-based installation
```

Click **Next**

---

### Step 3 — Select Server

Select the server from the pool:

```
Server Name : xitiztechservices.local
IP Address  : 192.168.1.220
Operating System : Windows Server 2025 Standard
```

Click **Next**

---

### Step 4 — Select WDS Role

Select:

```
Windows Deployment Services
```

The **Add Roles and Features Wizard** will appear.

Enable:

```
☑ Include management tools (if applicable)
```

Click:

```
Add Features
```

Then click:

```
Next → Next → Next
```

---

### Step 5 — Select Role Services

Enable the following services:

```
☑ Deployment Server
☑ Transport Server
```

Click **Next**

---

### Step 6 — Installation Confirmation

Enable:

```
☑ Restart the destination server automatically if required
```

When prompted:

```
Add Roles and Features Wizard → Yes
```

Click:

```
Install
```

⏳ Installation may take several minutes.

Once completed you will see:

```
Installation succeeded on xitiztechservices.local
```

Click **Close**

---

# ⚙️ Configure Windows Deployment Services

Open the WDS console.

```
Server Manager → Tools → Windows Deployment Services
```

Navigate to:

```
Windows Deployment Services
 └── Servers
      └── xitiztechservices.local
```

Right-click the server and select:

```
Configure Server
```

Click **Next**

---

# 🔗 Active Directory Integration

Select:

```
Integrated with Active Directory
```

Click **Next**

---

# 📂 Configure RemoteInstall Path

Specify the image storage directory.

```
D:\RemoteInstall
```

Click **Next**

Enable:

```
☑ Do not listen on DHCP and DHCPv6 ports
☑ Configure DHCP options for Proxy DHCP
```

Click **Next**

---

# 🖧 Configure PXE Response

Select the following options:

```
☑ Respond to any client computer
☑ Require administrator approval for unknown computers
```

Click **Next**

WDS will now complete the configuration.

Click **Finish** when completed.

---

# 📁 Verify RemoteInstall Folder

Open:

```
This PC → D:\
```

You should see a new folder automatically created:

```
D:\RemoteInstall
```

This folder stores deployment images.

---

# 💿 Add Windows 11 Install Image

Insert the **Windows 11 DVD**.

Example drive:

```
E:\
```

Navigate to:

```
Windows Deployment Services
 └── Servers
      └── xitiztechservices.local
           └── Install Images
```

Right-click:

```
Add Install Image
```

---

# 🗂 Create Image Group

Create an image group:

```
win 11 Pro
```

Click **Next**

---

# 📦 Select Install Image

Browse to:

```
E:\sources\install.wim
```

Click **Next**

Select only:

```
Windows 11 Pro
```

Leave all other editions **unchecked**.

Enable:

```
☑ Use the default name and description
```

Click:

```
Next → Next
```

WDS will import the image.

Once completed you will see:

```
The selected images were successfully added to the server
```

Click **Finish**

You should now see:

```
Install Images
 └── Windows 11 Pro
```

---

# ⚠️ Important Boot Image Note

Currently WDS works more reliably with **Windows 10 boot images** when deploying Windows 11.

Therefore we will use the **Windows 10 boot.wim** file.

---

# 💿 Add Boot Image

Insert the **Windows 10 DVD**.

Navigate to:

```
Boot Images
```

Right-click:

```
Add Boot Image
```

Browse to:

```
E:\sources\boot.wim
```

Click **Next**

---

# 🏷 Rename Boot Image

Set the following:

```
Image Name        : Microsoft Windows 11 Pro
Image Description : Microsoft Windows 11 Pro
```

Click **Next**

WDS will import the boot image.

After completion you will see:

```
The selected images were successfully added to the server
```

Click **Finish**

---

# 🔐 Configure PXE Client Response

Navigate to:

```
Servers → xitiztechservices.local
```

Right-click:

```
Properties
```

Open the **PXE Response** tab.

Enable:

```
☑ Respond to all client computers (known and unknown)
☑ Require administrator approval for unknown computers
```

Click:

```
Apply → OK
```

---

# 💻 Deploy Windows 11 to Client Computer

Go to the **client machine**.

Enter the **BIOS settings**.

Set the boot option to:

```
EFI Network Boot
```

The client will connect to the WDS server.

Example connection details:

```
Client IP  : 192.168.1.19
Server IP  : 192.168.1.220
Server Name: xitiztechservices.local
```

Press:

```
Enter
```

---

# ⏳ Administrator Approval

The client will display a message:

```
Waiting for administrator approval
Request ID : 1
```

Return to the **WDS server**.

Navigate to:

```
Windows Deployment Services
 └── Servers
      └── xitiztechservices.local
           └── Pending Devices
```

You will see a pending device with:

```
Request ID : 1
Status     : Pending
```

Right-click the device and select:

```
Approve
```

Confirmation message:

```
The pending device was approved successfully
```

Click **OK**

---

# 📥 Start Windows Installation

The client computer will automatically continue.

Click:

```
Next
```

Enter administrator credentials when prompted.

```
Domain Admin Username
Password
```

Click **OK**

---

# 🪟 Select Operating System

Choose:

```
Windows 11 Pro
```

Click **Next**

Continue installation as required.

---

# ✅ Deployment Completed

The system will install **Windows 11 through EFI Network Boot** using **Windows Deployment Services**.

This confirms successful **network-based OS deployment**.
