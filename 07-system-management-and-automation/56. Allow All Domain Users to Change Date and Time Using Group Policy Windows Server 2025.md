# 📘 Allow All Domain Users to Change Date and Time Using Group Policy - Windows Server 2025

## 📌 Overview

This guide explains how to allow **all domain users to change the system Date and Time** using **Group Policy (GPO)** in **Windows Server 2025**.

Normally, only administrators can change the system time. By configuring a **User Rights Assignment policy** in Group Policy, the IT department can allow domain users to modify the system time when required.

This configuration is useful in environments such as:

* 🖥️ Testing labs
* 🏢 Training environments
* 🔧 Specialized workstation configurations
* 🧪 Development environments

⚠ **Important:** Always test the policy before deploying it across the entire organization.

---

# ⚙️ Configuration Procedure

---

# 🪜 Step 1 — Open Group Policy Management

1. Open **Server Manager**.
2. Click **Tools**.
3. Select **Group Policy Management**.

This will open the **Group Policy Management Console (GPMC)** where administrators manage domain policies.

---

# 🪜 Step 2 — Create a New Group Policy Object (GPO)

Navigate to the **Organizational Unit (OU)** where the target users or computers are located.

### Example Environment

Domain:

```
xitiztechservices.local
```

---

## 📂 Create the Policy

1. Go to **Group Policy Objects**.
2. Right-click the OU.
3. Click:

```
Create a GPO in this domain, and link it here
```

4. Name the GPO:

```
Allow user to change Date and Time
```

5. Click **OK**.

---

## ✏️ Edit the Policy

1. Right-click the new GPO:

```
Allow user to change Date and Time
```

2. Click **Edit**.

---

## 🔎 Navigate to the Policy Setting

Go to the following path:

```
Computer Configuration
   └ Policies
      └ Windows Settings
         └ Security Settings
            └ Local Policies
               └ User Rights Assignment
                  └ Change the system time
```

Double-click:

```
Change the system time
```

---

## ⚙️ Configure the Policy

The **Change the system time Properties** window will open.

1. Tick mark the option:

```
Define these policy settings
```

2. Click:

```
Add User or Group
```

3. Click **Browse**

4. In the field:

```
Enter the object names to select (examples)
```

Type:

```
Authenticated Users
```

5. Click:

```
Check Names
```

6. Click **OK**

7. Click:

```
OK → Apply → OK
```

The policy is now configured.

---

# 🔄 Step 3 — Update Group Policy on Server

To apply the policy immediately on the server:

1. Open **PowerShell**.
2. Right-click **PowerShell**.
3. Select **Run as Administrator**.

Run the following command:

```powershell
gpupdate /force
```

This forces the server to refresh all Group Policy settings.

---

# 💻 Step 4 — Update Group Policy on Client Computers

Next, update the policy on the client machine.

1. Go to the **client computer**.
2. Open **PowerShell**.
3. Right-click → **Run as Administrator**

Run the command:

```powershell
gpupdate /force
```

4. Restart the computer.

The policy should now be applied.

---

# 🧪 Testing & Validation

⚠ Always test the policy before organization-wide deployment.

---

## ✅ Test 1 — Client Computer Verification

On the client computer:

1. Open **Control Panel**.
2. Go to:

```
Date and Time
```

3. Click:

```
Change date and time
```

If the policy is applied successfully:

✔ The user can now **change the date and time**.

---

# 📋 Best Practices

✔ Test the policy in a **test OU before production deployment**

✔ Verify that **Authenticated Users** are added correctly

✔ Ensure the **GPO is linked to the correct OU**

✔ Use **gpupdate /force** to immediately apply policy changes

✔ Restart client computers if the policy does not apply immediately


---

# 📂 Example Use Cases

This policy may be used in environments such as:

* IT labs
* QA testing systems
* Developer workstations
* Controlled training environments

⚠ In **production environments**, unrestricted time changes may affect:

* Domain authentication
* Kerberos tickets
* Logging and auditing
* Time synchronization services

---

# 📎 Summary

By following this guide, the IT department can configure a **Group Policy Object (GPO)** that allows **domain users to change system date and time** on Windows client computers in a **Windows Server 2025 Active Directory environment**.
