# 🔐 Block Specific Programs (.exe) for Domain Users Using Group Policy in Windows Server 2025

📁 **Category:** Active Directory / Group Policy

🖥 **Platform:** Windows Server 2025

👨‍💻 **Maintained by:** IT Department

📄 **Document Type:** Technical Implementation Guide


---

# 📖 Overview

This guide explains how to **block specific executable programs (.exe)** for **domain users** using **Group Policy** in a **Windows Server 2025 Active Directory environment**.

This method uses the **Group Policy setting "Don't run specified Windows applications"** to prevent users from launching restricted applications.

---

# 🎯 Objective

The goal of this configuration is to **restrict users from running specific applications** within the organization’s domain environment.

### Applications blocked in this example

| Application     | Executable    |
| --------------- | ------------- |
| Google Chrome   | `chrome.exe`  |
| Mozilla Firefox | `firefox.exe` |
| Microsoft Word  | `winword.exe` |

These applications will be **blocked for domain users** once the Group Policy Object (GPO) is applied.

---

# ⚙️ Prerequisites

Before starting, ensure the following requirements are met:

* Active Directory Domain Environment
* Windows Server 2025 Domain Controller
* Administrative privileges
* Target users located inside a **specific Organizational Unit (OU)**

Example domain used in this guide:

```
xitiztechservices.local
```

---

# 🪜 Implementation Steps

---

# 1️⃣ Open Group Policy Management

1. Open **Server Manager**.
2. Click **Tools**.
3. Select **Group Policy Management**.

This will open the **Group Policy Management Console (GPMC)**.

---

# 2️⃣ Create a New Group Policy Object (GPO)

Navigate to the **Organizational Unit (OU)** where the target users or computers are located.

### Steps

1. Go to **Group Policy Objects**.
2. Right-click and select **Create a GPO in this domain, and Link it here**.
3. Enter the GPO name:

```
Block Specific Programs (.exe)
```

4. Click **OK**.
5. Right-click the newly created policy:

```
Block Specific Programs (.exe)
```

6. Click **Edit**.

---

# 3️⃣ Configure Application Restriction Policy

Navigate to the following policy path:

```
User Configuration
 └ Policies
    └ Administrative Templates
       └ System
```

Locate the policy:

```
Don't run specified Windows applications
```

### Configuration Steps

1. Open **Don't run specified Windows applications**.
2. Select the radio button:

```
Enabled
```

3. Click the **Show** button under:

```
List of disallowed applications
```

4. Add the following executable names:

```
firefox.exe
chrome.exe
winword.exe
```

5. Click **OK**.
6. Click **Apply**.
7. Click **OK** again to save the policy configuration.

---

# 🔄 Update Group Policy on Server

After configuring the policy, update Group Policy to apply the changes.

### Steps

1. Open **PowerShell**.
2. Right-click **PowerShell**.
3. Select **Run as Administrator**.
4. Run the following command:

```powershell
gpupdate /force
```

This forces the server to **refresh all Group Policy settings**.

---

# 💻 Update Group Policy on Client Computer

The policy must also be refreshed on client machines.

### Steps

1. Go to the **client computer**.
2. Open **PowerShell**.
3. Right-click and select **Run as Administrator**.
4. Execute the following command:

```powershell
gpupdate /force
```

5. Restart the client computer.

# 🧪 Testing & Validation

⚠ **Important:** Always test policies before deploying them across the entire organization.

Testing ensures that the configuration behaves as expected and does not disrupt business operations.

---

## ✅ Test Example (AD Server or Client Machine)

1. Attempt to open **Google Chrome**.

Example:

```
chrome.exe
```

### Expected Result

A restriction notification message will appear:

> **This operation has been cancelled due to restrictions in effect on this computer. Please contact your system administrator.**

This confirms that the **Group Policy restriction is working correctly**.

---

# 📌 Best Practices

✔ Always test new GPO policies in a **test OU** before production deployment.

✔ Document all policy changes in your **IT change management system**.

✔ Ensure policies are applied only to **required users or groups**.

✔ Avoid blocking critical applications required for business operations.
