# 🖥️   Rename Local Administrator Account on Domain Computers via Group Policy (Windows Server 2025)

## 📘 Overview

This document provides a **step-by-step guide for renaming the Local Administrator account on domain-joined computers using Group Policy** in **Windows Server 2025**.

Renaming the default **Administrator** account is a common **security best practice**. Attackers often attempt to compromise systems by targeting the default administrator account name. Changing the name helps reduce this risk.

---

## 🎯 Objective

The purpose of this procedure is to:

* Standardize the **Local Administrator account name** across domain computers.
* Improve **security posture** by hiding the default administrator account.
* Deploy the configuration **centrally using Group Policy**.

---

# 🔎 Initial Verification (Client Computer)

Before applying the policy, verify the current local administrator account.

### Steps

1. Go to the **Client Computer**.
2. Open **Computer Management**.

```
Right Click Start Menu → Computer Management
```

3. Navigate to:

```
Local Users and Groups → Users
```

4. You will see multiple local user accounts.

Among them, you will find the default account:

```
Administrator
```

This is the account that will be renamed using **Group Policy**.

---

# 🪜 Step 1: Open Group Policy Management

1. Log in to the **Domain Controller**.
2. Open **Server Manager**.
3. Click:

```
Tools → Group Policy Management
```

This will open the **Group Policy Management Console (GPMC)**.

---

# 🪜 Step 2: Create a New Group Policy Object (GPO)

Navigate to the **Organizational Unit (OU)** that contains the computers where the policy should apply.

### Example Domain

```
xitiztechservices.local
```

### Steps to Create the GPO

1. Expand the domain in **Group Policy Management**.
2. Locate the appropriate **Organizational Unit (OU)**.
3. Right-click the OU.
4. Select:

```
Create a GPO in this domain, and link it here
```

5. Name the new Group Policy Object:

```
Rename Local Administrator Account On Domain Computers Via Group Policy
```

6. Click **OK**.

---

# ⚙️ Step 3: Configure the Administrator Rename Policy

1. Right-click the newly created GPO:

```
Rename Local Administrator Account On Domain Computers Via Group Policy
```

2. Click **Edit**.

3. Navigate to the following policy path:

```
Computer Configuration
   → Policies
      → Windows Settings
         → Security Settings
            → Local Policies
               → Security Options
```

4. Locate the policy:

```
Accounts: Rename Administrator account
```

5. Double-click the policy.

---

### Configure the Setting

A window will appear titled:

```
Accounts: Rename Administrator account
```

Perform the following:

* Enable the option:

```
Define this policy setting
```

* Enter the new administrator name:

```
Local-Admin
```

* Click:

```
Apply → OK
```

This configuration will rename the **default Administrator account** to **Local-Admin** on all targeted computers.

---

# 🔄 Step 4: Update Group Policy on the Server

After configuring the policy, update Group Policy on the **Domain Controller**.

### Steps

1. Open **PowerShell**.
2. Right-click **PowerShell**.
3. Select:

```
Run as Administrator
```

4. Run the following command:

```powershell
gpupdate /force
```

This forces the **Group Policy update** on the server.

---

# 💻 Step 5: Update Group Policy on the Client Computer

Next, update the policy on the **client machine**.

### Steps

1. Go to the **Client Computer**.
2. Open **PowerShell**.
3. Right-click and select:

```
Run as Administrator
```

4. Execute the command:

```powershell
gpupdate /force
```

5. Restart the computer.

The system will apply the updated Group Policy settings during reboot.

---

# 🧪 Testing & Validation

⚠ **Important:** Always test policies in a controlled environment before deploying them organization-wide.

---

## ✅ Test 1 — Verify on Client Computer

After restarting the client computer:

1. Open **Computer Management**.

```
Right Click Start Menu → Computer Management
```

2. Navigate to:

```
Local Users and Groups → Users
```

3. Review the list of local accounts.

You should now see:

```
Local-Admin
```

instead of:

```
Administrator
```

This confirms that the **Group Policy has successfully renamed the Local Administrator account**.

---

# 🔐 Security Best Practices

✔ Always rename the default administrator account

✔ Limit access to local administrator credentials

✔ Apply policies through **Organizational Units (OU)**

✔ Test policies before full production deployment
