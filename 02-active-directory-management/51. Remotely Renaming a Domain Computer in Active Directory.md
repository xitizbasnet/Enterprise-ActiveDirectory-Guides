## Remotely Renaming a Domain Computer in Active Directory

### Environment: Windows Server 2025

---

| Document Type       | IT Operations Procedure              |
| Target Audience     | IT Administrators / System Engineers |

| Server OS           | Windows Server 2025                  |
| Client OS           | Domain-Joined Windows Machine        |
| PowerShell Required | Yes                                  |
| Privilege Level     | Domain Administrator                 |

---

## 🎯 Purpose

This document provides a standardized procedure for remotely renaming a domain-joined computer using **Active Directory** and **Windows PowerShell** in a Windows Server 2025 environment.

---

## 🔍 Phase 1: Verify Current Client Computer Name

### 🖥️ On the Client Computer

1. Open client computer
2. Select **This PC** → Right click and select **Properties**
3. It will open **About** section
4. Now we can see the Device Name as:

```
Client-Win10
```

---

## 🏢 Phase 2: Rename the Computer from Active Directory

We will change this name from the Active Directory.

---

## 🛠️ Phase 3: Open Active Directory Server

### Step 1: Open Group Policy Management

1. Open **Server Manager**
2. Click **Tools** → **Group Policy Management**

---

### Step 2: Locate the Target Computer Object

📍 Navigate to the Organizational Unit (OU) where the target users are located.

### Example Environment

```
Domain: xitiztechservices.local
```

📂 Steps:

1. Go to **Group Policy Objects (Eg: Computers)**
2. We can see all the computers joined in this AD
3. Search for the computer Name as:

```
Client-Win10
```

---

## 💻 Phase 4: Rename Computer Using PowerShell

### 🔹 Generic Command Format

```powershell
Rename-Computer -ComputerName "Oldname" -NewName "Newname" -DomainCredential (Get-Credential) -Restart -Force
```

---

### 🔹 Command with Real Data

```powershell
Rename-Computer -ComputerName "Client-Win10" -NewName "TestSRV01" -DomainCredential (Get-Credential) -Restart -Force
```

---

## ▶️ Phase 5: Execute the Command

1. Open **PowerShell**
2. Run as an **Administrator**
3. Execute:

```powershell
Rename-Computer -ComputerName "Client-Win10" -NewName "TestSRV01" -DomainCredential (Get-Credential) -Restart -Force
```

---

### 🔐 Credential Prompt

* It will pop out the Windows PowerShell credential request user name and password
* Here write domain admin account credentials

---

## 🔄 Phase 6: Verify Name Change in Active Directory

Now refresh the:

Go to **Group Policy Objects (Eg: Computers)**

The computer name:

```
Client-Win10
```

Will change to:

```
TestSRV01
```

---

## 🔁 Phase 7: Automatic Restart

Client computer will restart automatically.

---

## ✅ Phase 8: Final Verification on Client Machine

### 🖥️ On the Client Computer After Restart

1. Open client computer
2. Select **This PC** → Right click and select **Properties**
3. It will open **About** section
4. Now we can see the Device Name as:

```
TestSRV01
```

---

# 🧾 Summary

| Step | Action                                     |
| ---- | ------------------------------------------ |
| 1    | Verify existing computer name              |
| 2    | Locate computer object in AD               |
| 3    | Execute Rename-Computer PowerShell command |
| 4    | Provide Domain Admin credentials           |
| 5    | System restarts automatically              |
| 6    | Verify new name in AD and client machine   |

---

# ⚠️ Important Notes

* Ensure the client machine is **online and reachable**
* Ensure you use correct **Domain Admin credentials**
* Confirm no duplicate computer name exists before renaming
* Inform users about automatic restart before execution
