# 🔐 Block InPrivate Browsing in Microsoft Edge using Group Policy in Windows Server 2025

This document provides a **step-by-step procedure** for disabling **InPrivate Browsing mode in Microsoft Edge** using **Group Policy (GPO)** in a **Windows Server 2025 Active Directory environment**.

Disabling InPrivate browsing helps organizations enforce **web usage policies, monitoring, and compliance** across domain-joined computers.

---

# 🏗 Environment Example

| Item      | Value                                        |
| --------- | -------------------------------------------- |
| Domain    | `xitiztechservices.local`                    |
| GPO Name  | `Block InPrivate Browsing in Microsoft Edge` |
| Target    | Client Computers (OU linked)                 |
| Server OS | Windows Server 2025                          |

---

# 📌 Overview

This guide will configure a **Group Policy Object (GPO)** that creates a **registry policy setting** to disable the **InPrivate browsing feature** in Microsoft Edge.

When applied:

✅ The **"New InPrivate Window" option becomes disabled (greyed out)**

✅ Users **cannot open private browsing sessions**

✅ Policy applies to **all domain computers within the linked OU**


---

# ⚙️ Implementation Procedure

---

# 🪜 Step 1 – Open Group Policy Management

1. Log in to the **Domain Controller** or **Windows Server 2025** with administrative privileges.

2. Open:

```
Server Manager
```

3. Navigate to:

```
Tools → Group Policy Management
```

This will open the **Group Policy Management Console (GPMC)**.

---

# 🪜 Step 2 – Create a New Group Policy Object (GPO)

Navigate to the **Organizational Unit (OU)** where the target **client computers** are located.

### Example Domain

```
xitiztechservices.local
```

---

## 📂 Create the GPO

1. Expand the domain.
2. Navigate to:

```
Group Policy Objects
```

3. Right-click → **New**

4. Enter the following name:

```
Block InPrivate Browsing in Microsoft Edge
```

5. Click **OK**

6. Right-click the newly created GPO:

```
Block InPrivate Browsing in Microsoft Edge
```

7. Click **Edit**

---

# 🛠 Step 3 – Configure Registry Policy

Inside the **Group Policy Management Editor**, navigate to:

```
Computer Configuration
   └ Preferences
        └ Windows Settings
             └ Registry
```

---

## Create New Registry Item

1. Right-click inside the **Registry panel**
2. Select:

```
New → Registry Item
```

---

## Registry Configuration

Configure the following values:

| Setting    | Value                            |
| ---------- | -------------------------------- |
| Action     | Update                           |
| Hive       | HKEY_LOCAL_MACHINE               |
| Key Path   | Software\Policies\Microsoft\Edge |
| Value Name | InPrivateModeAvailability        |
| Value Type | REG_DWORD                        |
| Value Data | 1                                |
| Base       | Decimal                          |

---

### Important Note

⚠ Do **NOT tick the "Default" checkbox** when entering the value name.

---

## Registry Path Structure

```
HKEY_LOCAL_MACHINE
 └ Software
     └ Policies
         └ Microsoft
             └ Edge
                 └ InPrivateModeAvailability
```

---

# 🔄 Step 4 – Update Group Policy on the Server

After configuring the policy, update Group Policy.

1. Open **PowerShell**

2. Run **as Administrator**

3. Execute the following command:

```powershell
gpupdate /force
```

This will **refresh all GPO policies on the server**.

---

# 💻 Step 5 – Update Group Policy on Client Computers

On the **target client computer**:

1. Open **PowerShell**

2. Right-click → **Run as Administrator**

3. Execute:

```powershell
gpupdate /force
```

4. Restart the computer.

This ensures the **new policy is applied to the system**.
