# 📘 Overview

In an enterprise environment, it may be necessary to prevent domain users from viewing the list of installed programs on their computers. This can help maintain system security, prevent unnecessary application awareness, and simplify the user interface.

This guide explains how to configure **Group Policy in Windows Server 2025** to hide the **Programs and Features** page from domain users.

---

# 🔍 Initial Verification (Client Computer)

Before applying the Group Policy, verify the current behavior on a client computer.

### Procedure

* First go to the client computer

* Go to the control panel

* Programs and Features

we can see the list of the program inside it

This confirms that the **Programs and Features** page is currently visible to users.

---

# ⚙️ Configuration Procedure

## 🪜 Step 1 — Open Group Policy Management

1. Open **Server Manager**.
2. Click **Tools**.
3. Select **Group Policy Management**.

📌 This console allows administrators to manage and configure Group Policy Objects within the Active Directory environment.

---

## 🪜 Step 2 — Create a New Group Policy Object (GPO)

Navigate to the **Organizational Unit (OU)** that contains the target users or computers.

### Example Domain

```
xitiztechservices.local
```

### Steps

1. Go to **Group Policy Objects**.
2. Right-click → **Create a GPO in this domain, and link it here**.
3. Enter the following name for the policy:

```
Hide Installed Programs from Domain Users in Program and features
```

Click Ok

4. Right-click **"Hide Installed Programs from Domain Users in Program and features"** → Click **Edit**.

---

## 🪜 Step 3 — Configure the Policy Setting

Navigate to the following path inside the **Group Policy Management Editor**:

```
User Configuration > Policies > Administrative templates > control panel > programs  > hide programs and features page > double click
```

it will pop up the "hide programs and features page"

tick mark on the: "Enabled" > apply > ok

📌 **Description**

Enabling this policy hides the **Programs and Features** page from the Control Panel. Users will not be able to view or manage installed applications.

---

# 🔄 Step 4 — Update Group Policy on Server

After configuring the policy, update the Group Policy on the server.

### Procedure

1. Open **PowerShell**.
2. Right-click → **Run as Administrator**.
3. Execute the following command:

```powershell
gpupdate /force
```

📌 This command forces the system to immediately apply updated Group Policy settings.

---

# 💻 Step 5 — Update Group Policy on Client Computer

Apply the updated Group Policy on the client machine.

### Procedure

1. Go to the client computer.
2. Open **PowerShell**.
3. Right-click → **Run as Administrator**.
4. Execute the following command:

```powershell
gpupdate /force
```

5. Restart the computer.

📌 Restarting ensures that the policy is fully applied to the user environment.

✔ Installed programs will not be visible

✔ Users will see the message:

```

```

# 📌 Best Practice Recommendations

* Always **test the GPO on a test OU** before production deployment.
* Apply the policy only to **required user groups**.* 
* Document any changes made to Group Policy in the IT change log.* 
* Maintain proper **GPO naming conventions** for easier administration.
